# SEO強化版 v1.8.0

## 目的

記事数だけを増やすのではなく、検索エンジンが理解しやすく、読者が購入・導入判断まで進みやすい構造へ変更します。全URLを保持しながら、根拠と検索意図が明確なページから段階的に検索対象へ出します。

## 現在の公開設計

- 全記事: 560件（日本語280件・英語280件）
- 検索対象: 244件（日本語122件・英語122件）
- 段階待機: 316件（`noindex,follow`）
- データ生成記事: 478件
- 検索対象記事の固有画像: 244枚
- 目的別テーマ: 12テーマ×2言語、一覧を含め26ページ
- サイトマップ掲載URL: 297件

検索対象は、既存公開URL、アフィリエイトへつながる購入意図記事、優先製品レビュー、優先製品同士の比較を基準に選びます。日英の片方だけを公開対象にしないよう、`translationKey` 単位でそろえます。

## 強化した項目

1. 各記事の冒頭に判断要約を置き、結論と確認項目を先に示します。
2. 公式情報の件数、最終確認日、目的別ガイドを検証情報として表示します。
3. 検索対象記事ごとに固有の1200×675 JPEGを生成し、本文、OG、Twitter、JSON-LD、画像サイトマップで共通利用します。
4. 関連記事をカテゴリ順ではなく、トピック、製品、記事種別から選びます。各データ生成記事は5件へリンクし、受信リンクの偏りも抑えます。
5. 12の目的別ハブを追加し、ホーム、カテゴリ、記事、記事一覧から移動できるようにします。
6. 長い検索タイトルと説明文を言語別の上限へ自動調整します。
7. 日英の `hreflang` と `x-default` を対応する日本語URLへ統一します。
8. サイトマップを固定ページ、日英記事、日英トピックへ分割し、全URLへ `lastmod` を付けます。
9. 日英RSSとIndexNow通知を生成します。

## 段階公開の変更方法

`content/config/seo.json` の `indexing` を変更します。

- `mode: "priority"`: 現在の推奨設定。優先記事だけを検索対象にします。
- `includeSourceFiles`: 特定バッチの記事を日英まとめて追加します。
- `priorityProducts`: 収益性と事業展開に近い製品を追加します。
- `mode: "all"`: 全記事を検索対象にします。固有の一次検証と検索意図の整理が完了した後だけ使用します。

変更後は必ず次を実行します。

```bash
npm run check
```

検証は、リンク切れ、日英対応、noindex混入、固有画像、構造化データ、トピック件数、検索データ、サイトマップ、メタ情報の重複を確認します。

## Search Consoleで行うこと

1. `https://luqevora.com/sitemap.xml` を送信します。
2. 主要なホーム、トピック、比較、レビューをURL検査します。
3. 「クロール済み - インデックス未登録」と検索パフォーマンスを週次で確認します。
4. 表示回数はあるがクリック率が低い記事のタイトルと説明文を改善します。
5. noindex記事は、固有の検証結果、画面例、比較軸を追加してから優先設定へ移します。

IndexNowはGitHub Pagesの本番公開後に自動通知します。Google向けのインデックス登録を保証するものではないため、Search ConsoleのサイトマップとURL検査を主軸に運用します。
