Webflow Review: Starter, Site and Workspace Plans, CMS and Code Export

This review organizes Webflow’s free Starter Site, Site and Workspace plans, custom domains, CMS, backups, code export and Ecommerce using official documentation only. It does not independently rank usability or design quality.

Published and checked: July 16, 2026Estimated reading time: 11 minutesBy Luqevora.com Editorial Team

Method: We reviewed official pricing, help, security and migration documentation on July 16, 2026. Prices, plans, availability and features can change, so reconfirm them at checkout.

Scope: This is an official-source review for purchase decisions. We did not perform a reproducible usability or performance test, so it does not rank ease of use, speed or design quality.

Conclusion: a candidate for teams that need granular visual design and structured CMS management

Decision summary

Webflow

Webflow combines visual site design, CMS and managed hosting. You can start on the free Starter Site plan, but publishing to a custom domain requires a paid Site plan. Collaboration and code export can also depend on the Workspace plan, so Site and Workspace subscriptions should be evaluated separately.

Advantages confirmed in official documentation

  • Start building and publish to a webflow.io staging domain on the free Starter Site
  • Use a paid Site plan for custom-domain hosting and form submissions
  • Use Premium for CMS, site search, file uploads and content-rich sites
  • Export HTML, CSS, JavaScript and assets with an eligible paid Workspace plan

Constraints to check before subscribing

  • Site and Workspace plans cover different capabilities and may both need review
  • The free Starter Site is limited to two static pages and 50 CMS items
  • Code export does not reproduce Webflow’s dynamic CMS, and exported forms need another submission method
  • Native checkout requires a separate Ecommerce Site plan

Code can be exported, but Webflow’s dynamic services do not become a complete portable clone

Static code

Eligible paid Workspace plans can export HTML, CSS, JavaScript and assets for external hosting without Webflow attribution.

CMS and forms

Dynamic content must be exported collection by collection. Webflow form handling stops on an exported site, so a custom action or third-party form service is required.

Operational note: Before choosing external hosting, define replacements for CMS, forms, search, Ecommerce, memberships and redirects.

Who may fit and who should compare alternatives

May be a fit

  • Designers need detailed control over layout and CMS structure
  • A company site, case studies, careers and editorial content should share one system
  • Agencies and internal editors need separate roles
  • Static code export should remain an option

Compare alternatives

  • The only goal is to launch a minimal site as quickly as possible
  • You want to avoid separate Site and Workspace subscription decisions
  • You expect exported code to retain Webflow CMS and form services
  • Complex inventory and commerce operations are the primary requirement

Pre-purchase checklist

  1. 1. Separate Site-plan requirements from Workspace-plan requirements
  2. 2. Check page, CMS, bandwidth, form and file-upload limits
  3. 3. Define seats and publishing permissions
  4. 4. Plan replacements for CMS, forms and search if exporting
  5. 5. For Ecommerce, confirm item limits, fees and payment availability

Frequently asked questions

Can the free Starter Site use a custom domain?

No. Publishing to a custom domain requires a paid Site plan.

Can a Webflow site be hosted elsewhere?

Eligible paid Workspace plans can export HTML, CSS, JavaScript and assets. CMS data must be exported separately, and Webflow form processing does not continue automatically on the exported site.

What is the difference between a Site plan and a Workspace plan?

A Site plan covers publishing, hosting, CMS and site-level features. A Workspace plan covers the build environment, staging, collaboration and capabilities such as code export.
