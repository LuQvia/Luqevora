KinstaへWordPressを移行する方法｜無料移行・DNS切替・公開前チェックリスト

KinstaはWordPressサイトの無料移行を案内しています。申請前の準備、移行中の更新停止、DNS切替、公開後確認を順番に整理します。

公開・最終確認：2026年7月24日読了目安：10分執筆：Luqevora.com編集部

移行の要点

サイトコピーと公開切替を分ける

Kinstaはプラン内のWordPressインストール数を上限として無料移行を案内しています。移行完了後に旧サイトへ注文や投稿が入ると差分が生じるため、更新停止時間と最終同期方法を事前に決めます。

移行申請前に準備する情報

  • 現在のホスティング会社と管理画面
  • WordPress管理者アカウント
  • SFTP、SSH、データベース情報
  • DNSを管理するサービスとログイン情報
  • キャッシュ、WAF、CDN、リバースプロキシの有無
  • メールサーバーが同じ契約に含まれているか
メールの注意：KinstaへのWebサイト移行とメール移行は別です。現行サーバーの解約前にMXレコードとメール保存先を確認します。

移行から公開までの手順

  1. 1.MyKinstaで移行を申請し、認証情報を安全に共有する
  2. 2.移行先の一時URLで表示、管理画面、PHP、プラグインを確認する
  3. 3.EC・会員サイトは更新停止または差分移行の時間を決める
  4. 4.DNS TTLを事前に短くし、切替時刻を関係者へ共有する
  5. 5.DNSを切り替え、SSL、リダイレクト、www有無を確認する
  6. 6.フォーム、決済、メール通知、計測、検索クロールを確認する

EC・会員・予約サイトは差分データを優先

移行作業中も旧サイトで注文、予約、会員登録、コメントが発生すると、最初のコピーには含まれません。更新停止時間を設けるか、公開直前に差分を反映する方法をKinstaへ確認します。

静的な企業サイト

  • 短時間の更新停止を設定しやすい
  • 公開前確認を十分に行いやすい
  • 差分が少ない

動的サイト

  • 注文・予約・会員データの差分が重要
  • キャッシュとセッションの確認が必要
  • 切替時間帯を慎重に選ぶ

よくある質問

Kinstaへの移行は無料ですか？

Kinstaは無料・無制限のWordPress移行を案内していますが、利用可能なWordPressインストール数が実質的な上限です。

DNS切替までKinstaに任せられますか？

移行作業とDNS管理は分けて考えます。現在のDNS管理会社と切替権限を事前に確認してください。

旧サーバーはいつ解約しますか？

DNS伝播、フォーム、メール、決済、計測、バックアップを確認した後に解約します。
